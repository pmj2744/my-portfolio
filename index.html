기말고사 프로젝트

SecureWeb - 보안 웹 애플리케이션 구현 및 취약점 검증

중간고사 때 조사한 웹 보안 이론을 바탕으로, 실제 로그인·회원가입 기능과 커뮤니티 게시판을 갖춘 웹 서비스를 구현하고 OWASP Top 10 기반의 보안 취약점을 방어 및 검증한 턴키 프로젝트입니다.

프로젝트 주제
안전한 웹 서비스 구현 및 모의 해킹을 통한 웹 취약점(SQL Injection, XSS) 방어 대책 검증
주요 내용
- **개발 및 구현**: HTML5, CSS3, JavaScript를 활용한 프론트엔드 UI/UX 설계 및 Node.js 기반 백엔드 연동
- **보안 기능**: 구글 캡차(reCAPTCHA) 우회 방지, 정규표현식 기반의 입력값 필터링, 비밀번호 단방향 암호화 적용
- **취약점 분석**: 무차별 대입 공격(Brute Force), SQL 인젝션, 크로스 사이트 스크립팅(XSS) 모의 해킹 시연 및 시나리오 작성
팀원 소개
총 3명의 팀으로 프로젝트를 진행하였으며, 각자 역할을 나누어 개발을 진행했습니다.
[팀원 상세 역할 더 보기]
총 3명의 팀원으로 구성되어 역할 분담 후 협업을 진행했습니다.
- 박민지: 실시간 현재 시간 출력, Security 퀴즈 시스템 구현
- 박시현: 동적 이미지 슬라이더, 정보 검색창 + 검색어 기능 구현
- 김정은: 카테고리 필터링, 마우스 이벤트 기반 Security 요약 모달창 기능 구현
내 역할 (팀장)
팀장으로서 기말 프로젝트의 전체 일정을 조율하고 보고서 작성을 총괄했습니다. 기술적으로는 수업 시간에 배운 자바스크립트를 활용하여 구현했습니다. 그리고 중간 브릿지 역할을 수행했습니다.
